Version 1.02 (8/18/2016)

  • Mod support: Improved database import. Entries to import are identified by their template, not path.

Version 1.01 (7/26/2016)

  • Improved database import to require less memory, still requires a 64 bit JVM (due to a bug in Oracle's 32 bit JVM)
  • Bugfixes in support for Spears and Staffs


Version 1.00 (7/18/2016)

  • Mod support: A mod can now be specified on the config page. Each mod needs to be imported separately and stores its items separately to avoid conflicts.
  • Mod support: a stash now retains all items and an item retains its attributes (e.g. prefix), even if the currently loaded database (vanilla / specific mod) does not support that item.
    If the item is unknown, it will not be shown. If an attribute of the item is unknown, the attribute will not be shown but will be kept.
  • Items now have a transparent background, which allows them to be colored by rarity
  • Support for the current 1.0.0.5 format added


Version 0.99g (6/5/2016)

  • minor bugfixes


Version 0.99d (5/31/2016)

  • 'Char Editor' screen: Faction Reputation can be edited
  • Stashes and Characters from mods can be loaded
    The tool still only has the vanilla items, so only use this if you have no mod-specific items, as they will get lost
  • 'Configuration' screen: New setting to specify whether to copy or move items from the tool's DB to the stash / char
  • The tool's data is now stored in the user directory, not the program directory, to avoid conflicts with Window's restrictions
  • The tool's configuration is now saved in an ini file (before it was in the tool's database), part of ongoing work for mod-support
  • 'Char Editor' screen: The stat increments when pressing the +/- buttons are now taken from database.arz, part of ongoing work for mod-support
